#Q-Sharp #Quantum #Math
**Rotation gates** are parameterized single-qubit unitaries rotating the qubit state on the [[Bloch sphere]] by a continuous angle $\theta$. All are in `Std.Intrinsic` & support `Adj + Ctl`.

**`Rx(θ, q)` - rotation around X-axis**
$$R_x(\theta) = e^{-i\theta X/2} = \begin{pmatrix}\cos\frac{\theta}{2} & -i\sin\frac{\theta}{2}\\-i\sin\frac{\theta}{2} & \cos\frac{\theta}{2}\end{pmatrix}$$
$R_x(\pi) = -iX$ (X gate up to global phase). Adjoint: $R_x(-\theta)$.
```csharp
Rx(PI() / 2.0, q);        // 90° rotation around X
Adjoint Rx(theta, q);     // equivalent to Rx(-theta, q)
```

**`Ry(θ, q)` - rotation around Y-axis**
$$R_y(\theta) = e^{-i\theta Y/2} = \begin{pmatrix}\cos\frac{\theta}{2} & -\sin\frac{\theta}{2}\\\sin\frac{\theta}{2} & \cos\frac{\theta}{2}\end{pmatrix}$$
Real-valued [[Matrix]] - no complex entries. $R_y(\pi) = -iY$. Used in variational circuits ([[VQE]], [[VQA]]) as the standard parametric gate.
```csharp
Ry(theta, q);
Controlled Ry([ctrl], (theta, q));    // controlled rotation
```

**`Rz(θ, q)` - rotation around Z-axis**
$$R_z(\theta) = e^{-i\theta Z/2} = \begin{pmatrix}e^{-i\theta/2} & 0\\0 & e^{i\theta/2}\end{pmatrix}$$
Applies phases $e^{\mp i\theta/2}$ to $|0\rangle$ & $|1\rangle$. $R_z(\pi) = -iZ$.
```csharp
Rz(theta, q);
```

**`R1(θ, q)` - phase rotation**
$$R_1(\theta) = \begin{pmatrix}1 & 0\\0 & e^{i\theta}\end{pmatrix}$$
Leaves $|0\rangle$ unchanged; applies phase $e^{i\theta}$ to $|1\rangle$. Differs from $R_z$ by a global phase: $R_1(\theta) = e^{i\theta/2} R_z(\theta)$. Used extensively in QFT & [[QPE]].
```csharp
R1(2.0 * PI() / 4.0, q);                   // phase π/2 to |1⟩  (= S gate)
R1(PI(), q);                                // phase π to |1⟩    (= Z gate)
Controlled R1([ctrl], (theta, q));          // controlled phase - CRk gate in QFT
```
**Special values**: $R_1(\pi/4) = T$, $R_1(\pi/2) = S$, $R_1(\pi) = Z$.

**`R(pauli, θ, q)` - general Pauli rotation**
$$R(P, \theta) = e^{-i\theta P/2} = \cos\frac{\theta}{2}\cdot I - i\sin\frac{\theta}{2}\cdot P \quad P \in \{I, X, Y, Z\}$$
Generalizes all $3$ axis rotations. `PauliI`, `PauliX`, `PauliY`, `PauliZ` are the Q# enum values.

```csharp
R(PauliX, theta, q); // same as Rx(theta, q)
R(PauliZ, theta, q); // same as Rz(theta, q)
```

**`Exp(paulis, θ, qs)` - multi-qubit Pauli exponential**
$$\text{Exp}(P_1 \otimes \cdots \otimes P_n,\, \theta) = e^{-i\theta (P_1\otimes\cdots\otimes P_n)/2}$$
Applies rotation generated by a [[Tensor]] product of Paulis across multiple [[Qubits]]. Used in Trotterization for Hamiltonian simulation.
```csharp
Exp([PauliZ, PauliZ], theta, [q0, q1]);    // e^{-iθZZ/2}
```

**Adjoint of rotation gates**

All rotation gates satisfy `Adjoint Rx(θ, q) = Rx(-θ, q)`. Q# auto-generates the adjoint by negating the angle parameter.
```csharp
Adjoint Ry(theta, q);     // rotation by -theta
```
This makes uncomputing rotation-based circuits straightforward - no new gate needed, just negate the angle.
